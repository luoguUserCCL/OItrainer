## 一、严重Bug（阻塞性）

### Bug 1: real-render.js 语法错误 ✅ 已修复
- **位置**: `lib/real/real-render.js` 第216行附近
- **问题**: `_renderStudents` 方法中的 `for` 循环缺少闭合的 `}`，导致整个 `RealRender` 模块无法加载，游戏界面完全空白
- **修复**: 添加了 `} /* end for students */`
- **影响**: 游戏完全无法运行

### Bug 2: 训练行动永远不生效（action name 不匹配）
- **位置**: `lib/real/real-render.js` 第688行
- **问题**: 渲染代码调用 `executeAction('训练', ...)` 但 `REAL_ACTIONS` 中的 key 是 `'做题训练'`
- **结果**: 所有训练行动返回"未知行动: 训练"，学生永远不会获得知识增长
- **影响**: 核心玩法完全失效

### Bug 3: "对点招生"和"请学生吃饭"行动不存在
- **位置**: `lib/real/real-render.js` 第891、898行
- **问题**: 渲染代码调用 `executeAction('对点招生', ...)` 和 `executeAction('请学生吃饭', ...)`，但这两个 action 在 `REAL_ACTIONS` 中不存在
- **结果**: 点击这两个按钮始终失败

### Bug 4: 比赛周无法推进
- **位置**: `lib/real/real-game.js` `advanceWeek()` 方法
- **问题**: 当到达比赛周（如CSP-S1第3周）时，`advanceWeek` 返回 `'contest_week'` 但不推进周数。如果比赛UI未正确显示/处理，玩家将永久卡在该周
- **影响**: 游戏流程可能卡死

### Bug 5: 比赛日志显示 [object Object]
- **问题**: 比赛模拟器的 log 数组存储的是对象而非字符串，导致UI显示 `[object Object]`

### Bug 6: 省份选择页面显示数字而非名称
- **位置**: `real-start.html` 第351行
- **问题**: 省份卡片使用 `key`（数字 1-33）作为显示文本，而非 `PROVINCES[key].name`（中文名称）

## 二、平衡性问题

### 问题1: 比赛分数方差过大
- **现象**: 同一个学生在同一场比赛中，10次运行得分范围极大
  - 吴桓: 0~237分 (方差237)
  - 罗湧: 80~380分 (方差300)
  - 冯博之: 0~275分 (方差275)
- **分析**: tick-based模拟器的随机性导致学生有时选择困难题目浪费所有时间，有时连续做对简单题
- **建议**: 降低随机性，加入"优先选择已部分完成的题目"等策略

### 问题2: 知识点对比赛得分影响过小
- **现象**: 学生所有知识点都为E级(15)，却能获得200+甚至400分的比赛成绩
  - 黄明承(只有math=25, 其余全15) 获得400/400满分
  - 这意味着demand系统没有有效限制得分
- **分析**: `_calcSpeedMultiplier` 和 `_selectSubtask` 似乎主要由 thinking/coding 决定，knowledge 只是辅助因素
- **建议**: 大幅增加 knowledge 对解题能力的权重，使 E 级学生在高 demand 题目上几乎无法得分

### 问题3: 训练增益合理但任务覆盖不均
- **训练增益（中等强度）**:
  - 知识: +5~10（取决于任务和知识领域）
  - 思维: +2~4
  - 代码: +1~3
  - 体力消耗: 8~15
  - 压力增加: +10~15
- **问题**: weeklyTasks 随机选择，可能连续数周都缺少某些知识领域的题目（如 graph、DP），导致学生发展极不均衡
- **建议**: 确保 weeklyTasks 中涵盖所有知识领域

### 问题4: 集训(Camp)效率过高
- **现象**: 一次集训同时提升 DS+3~8, Graph+7~11, String+5~6, Math+3~6, DP+5~8
- **对比**: 单次训练只提升一个领域 +3~5
- **建议**: 降低集训增益或增加成本/副作用

### 问题5: CSP-S1 配置问题
- **当前**: 4题, totalMaxScore=400, diff=[40,55,70,85]
- **历史**: 之前已修复为1题/100分/diff=60，但似乎被还原
- **建议**: 确认期望的 CSP-S1 格式

### 问题6: 学生初始数值差异过大
- **现象**: 同一批学生的 thinking 差距可达2倍（36 vs 76），coding 差距可达2倍（29 vs 63）
- **影响**: 天赋差距过大导致某些学生几乎无法培养
- **建议**: 缩小初始值范围
